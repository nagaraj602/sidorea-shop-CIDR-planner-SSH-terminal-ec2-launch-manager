AWS Storage Levels & EBS Configuration

1. Storage Levels Architecture

Storage levels dictate how data is structured (block, file, object), decided based on file organization and metadata capability.

📁 File Level Storage (EFS / FSx)

  • Stores data in a top-bottom hierarchy (root ➔ bin/home ➔ files).
  • Stores metadata partially. Not bootable devices.
  • Uses NFS protocol (EFS for Linux) or SMB (FSx for Windows).
  • Can be mounted simultaneously on multiple EC2 instances.
  • Designed for shared file systems and distributed workloads across multiple AZs.

🧱 Block Level Storage (EBS)

  • Divides data into fixed-size independent chunks (blocks) like 4kb, 16kb.
  • Does not store metadata; remembers the index value of chunks.
  • Attached to an EC2 instance as a mounted disk (e.g., /dev/xvda) using the OS file system.
  • Limited to a single AZ. Offers very low latency and high IOPS for boot volumes and databases.

🪣 Object Level Storage (S3)

  • Flat structure. Data stored as objects (file + full metadata) inside buckets.
  • Accessed via HTTP/HTTPS APIs (URL), not mounted as a traditional disk.
  • Not a bootable device. Highly durable (11 9's).
  • Optimized for durability, data lakes, and static hosting rather than real-time latency.

3. Runbook: Attaching a New EBS Volume (Without Rebooting)

After creating the volume in AWS Console (EC2 -> Volumes -> Attach Volume to /dev/sdb or /dev/xvda), follow these Linux OS steps:

# 1. List block devices to verify the disk is attached but unmounted $ lsblk # 2. Check the file system of the new device (returns 'data' if empty) $ sudo file -s /dev/xvdc # 3. Create a filesystem on the EBS volume (Linux: xfs, ext3, ext4) $ sudo mkfs -t xfs /dev/xvdc # 4. Create a mount point directory $ sudo mkdir /demo # 5. Mount the device to the directory $ sudo mount /dev/xvdc /demo # 6. Verify it is mounted successfully $ df -Th # 7. Make it persistent on reboot by grabbing the UUID $ sudo blkid /dev/xvdc # 8. Add the entry to the fstab file using UUID (Recommended) or Device Name $ sudo vi /etc/fstab # Inside vi, append this line: UUID=a1234566789 10f847894 /demo xfs defaults,nofail # OR using device name: /dev/xvdc /demo xfs defaults,nofail
