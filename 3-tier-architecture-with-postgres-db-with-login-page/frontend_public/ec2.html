EC2 Instance Types

EC2 Naming Suffix Decoder (Example: m7g.large)
• m : Family (General Purpose)
• 7 : 7th Generation
• g : Graviton Processor (a = AMD, i = Intel)
• d : Instance Storage (Ephemeral NVMe SSD - highly fast, different from EBS)

General Purpose Instances

Provide a balance of compute, memory, and networking resources. Used for webservers, code repositories, and small-to-medium databases.

M Series (Production)

M7gM6iM5
  • Balanced resources, good for small to medium database, enterprise applications, webservers.
  • Considered the standard for production works in an organization.

T Series (Dev/Test)

T4gT3T2
  • Burstable performance instances. Provide a baseline CPU performance with the ability to "burst" above the baseline using CPU credits.
  • Ideal for applications with variable/low-to-moderate CPU usage (small web servers, microservices, CI/CD pipelines).
  • Typically used for development & testing environments.

Compute Optimized Instances

Ideal for compute-bound applications that benefit from high-performance processors.

C Series

C Series
Use Cases: Batch processing, gaming servers, machine learning inference, media transcoding.

Memory Optimized Instances

Designed to deliver fast performance for workloads that process large data sets in memory.

R, X, and Z Series

R Series (R8g, R7g, R6g, R5) X Series (X2gd, X1e) Z Series (Z1d)
Use Cases: Data warehouse applications (requires fast processing), high-performance databases, big data analytics, real-time processing.

Note on Instance Storage (d): Some instances (like X2gd) feature Ephemeral Storage. It is highly fast NVMe SSD storage located on the host hardware (excellent for weather forecast applications), but it is volatile and different from persistent EBS.

Storage Optimized Instances

Deliver millions of low-latency, random I/O operations per second. Designed for workloads requiring high, sequential read/write access to massive local datasets.

I, D, and H Series

I Series (I4i, I3en, I3) D Series (D3en, D2) H1 Series
Use Cases: Big data applications (e.g., Instagram backend), high-throughput databases, log processing, data streaming.

Accelerated Computing (GPU)

Use hardware accelerators (co-processors) to perform functions more efficiently, such as floating point number calculations or graphics processing.

P, G, and F Series

P Series (P3) G Series (G4) F Series (F1 - FPGAs)
Use Cases: Machine learning, graphics rendering, scientific simulations, specialized pattern matching.

High Performance Computing (HPC)

Offers the best price performance for running HPC workloads at scale. Ideal for complex simulations, deep learning, and VFX rendering.

HPC Series

HPC8a HPC7a HPC7g HPC6id HPC6a

Reminder on Suffixes: a = AMD, g = Graviton, i = Intel, d = Instance (ephemeral) storage.
